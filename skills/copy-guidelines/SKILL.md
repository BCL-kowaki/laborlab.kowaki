# SKILL: コピー運用ルール

## 目的

本プロジェクトで扱うすべての日本語コピーについて、**一貫性・正確性・ブランド性**を維持するためのルール集。

---

## ⚡ 基本原則

1. **コピーの出典は常に `content/copy.md`**
   コード内やコンポーネントに直接日本語コピーを書かない。すべて `content/copy.md` から取得する。

2. **固有名詞の匿名化ルールを厳守**
   特定のブランド名・サービス名・社名を絶対にサイト上に出さない（後述）。

3. **トンマナの一貫性**
   「信頼感 × テック感 × 親しみやすさ」の3軸でバランスを取る。

4. **変更時は必ず `content/copy.md` を先に編集**
   コードを先に変えてから `copy.md` を後で追従する、という順序は厳禁。

## 🏷 ブランド階層の遵守

```
【屋号（正式名称）】 小脇社会保険労務士事務所
【スローガン】      Labor Lab.
【タグライン】      新しい"攻め"の労務で、経営を伴走する。
```

### 表記ルール

| 場面 | 使う表記 |
|---|---|
| 本文・公式書類 | 「小脇社会保険労務士事務所」 |
| 省略形（口語・SNS） | 「小脇社労士事務所」 |
| ロゴ | 「小脇社会保険労務士事務所」＋ 小さく `Labor Lab.` |
| ビジュアルアクセント | `Labor Lab.` 単独OK |
| SNS・広告 | `Labor Lab.` をキャッチ的に使用可 |
| ❌ NG | 「小脇社労士事務所 ／ Labor Lab」の並列表記 |
| ❌ NG | 「Labor Lab株式会社」「Labor Lab事務所」等、屋号扱い |

### スローガンの扱い

`Labor Lab.` は**屋号ではなくキャッチコピー**です：

- 文章中に自然に挿入するのは避け、**ビジュアル主体**で使用
- 代表挨拶・FAQ・料金表など本文コンテンツで頻繁に使わない
- 定義文を複数箇所に書かない（説明は1箇所で十分）
- 法人登記・公式書類への使用は避ける

## ✍️ トーン & スタイル

### 文体

- **です・ます調**で統一（常体は使わない）
- 堅すぎず、崩れすぎない品のある語り口
- 業界用語は必要に応じて使うが、初出時に簡潔な補足

### ❌ 使わない表現

- ❌ 「必ず〇〇できます」「絶対に〇〇」等の断定的な保証表現
  → ✅ 「〇〇を目指します」「取りこぼしをゼロに近づけます」
- ❌ 「業界No.1」「日本一」等の根拠のない最上級
  → ✅ 「鹿児島で唯一の」等、**事実ベースの差別化**のみ
- ❌ 「弊社」「当社」
  → ✅ 「当事務所」
- ❌ 「お客様」の濫用
  → ✅ 「経営者様」「顧問先」「貴社」を場面で使い分け
- ❌ 絵文字の多用（ただし📌🥇🥈🥉は箇所を絞ってOK）

### ✅ 推奨する表現

- 問いかけ形式: 「〜していませんか？」「〜、心当たりありませんか？」
- 数字・固有情報: 「1人あたり最大80万円」「2025年4月の法改正」等
- 対比: 「〜する会社と、〜しない会社」
- 経営ワード: 「本業に集中」「経営を前に進める」「取りこぼし」

### 改行・読点

- 長い文は適度に改行して読みやすく
- 読点（、）は詰めすぎず、息継ぎ位置に
- コピー内の改行は原則 `<br />` ではなく段落分け（`<p>`）で対応

---

## 📏 文字装飾のルール

### 強調

- `**太字**`: コピー内で最重要の1〜2箇所のみ
- 「鍵括弧」: 固有名詞・概念語（例: 「Labor Lab.」「助成金レーダー」）
- _斜体_: 欧文のみ、和文には使わない

### リスト

- 📌 リスト頭のアイコン: 箇条書きの代わりに使うとAIDERっぽい
- 🥇🥈🥉 ランキング感の表現: 3項目までの重要度表現
- `・` や `-` の素のリスト: 情報が多い時のみ使用

### 数字の扱い

- 半角数字を使用（「3万円」ではなく「¥30,000」）
- 料金: `¥35,000`（税別）のように単位を明確に
- 人数: `1名`, `10名`, `50名以上` のように「名」

## 📦 コピーの取り回し（実装観点）

### lib/copy.ts での定数化

`content/copy.md` から抽出したコピーは、`src/lib/copy.ts` に定数として配置：

```typescript
// src/lib/copy.ts
export const HERO = {
  mainHeading: ['損してない？', 'もらえるはずの助成金を', '見過ごしてませんか？'],
  subHeading: '「労務 × AI」新しい形。\n新しい"攻め"の労務で、\nあなたの経営を伴走します。',
  description: '小脇社会保険労務士事務所は、...',
  ctaPrimary: '無料相談を申し込む',
  ctaSecondary: '資料をダウンロード',
} as const;

export const SLOGAN = 'Labor Lab.' as const;

export const BRAND = {
  name: '小脇社会保険労務士事務所',
  shortName: '小脇社労士事務所',
  slogan: 'Labor Lab.',
  tagline: '新しい"攻め"の労務で、経営を伴走する。',
} as const;
```

### JSXでの使用

```tsx
import { HERO, BRAND } from '@/lib/copy';

<h1>
  {HERO.mainHeading.map((line, i) => (
    <span key={i} className="block">{line}</span>
  ))}
</h1>
```

### ❌ 絶対にやらない

```tsx
// ❌ BAD: コピーを直書き
<h1>損してない？</h1>

// ❌ BAD: コンポーネント内で文言を変える
<h1>{HERO.mainHeading[0] + '？'}</h1>
```

## 📝 コンテンツ更新フロー

コピー変更が必要になった場合の手順：

1. Takuyaさんに変更内容を確認
2. `content/copy.md` を編集（変更理由をコミットメッセージに明記）
3. `src/lib/copy.ts` の対応する定数を更新
4. コンポーネントで使用している場所を確認・更新（通常は不要）
5. `pnpm build` でビルドチェック
6. プレビュー環境で確認後、本番反映
